DescriptionÔÉÅ

This repository contains python modules that can be used together with NOVA-Server.

Repository structureÔÉÅ

The structure of the repository resembles the folder structure of the NOVA CML folder. In principle, we distinguish between chains and models. In the terminology of NOVA-Server a chain will always be implemented as a Extractor while a model will always be implemented as either a Trainer and/or and Predictor:

  • Extractor: Stored in the chains directory. Called by using the /extract route of the server. Writes a data stream to disk.

  • Predictor: Store in the trainer/trainer directory. Called by using the /predict route of the server.

  • Trainer: Stored in the trainer/template directory. Called by using the /train route of server. Trains a model and writes the trained model to the template directory. A trainer is always also a predictor.
