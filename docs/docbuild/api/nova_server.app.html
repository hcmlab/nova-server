nova_server.app module

Standalone script for data predictions

Author:

Dominik Schiller <dominik.schiller@uni-a.de>

Date:

06.09.2023

This script performs predictions for annotations in the NOVA database using a provided nova-server module for inference and saves the results to the NOVA-DB.

param –env:

Path to the environment file to read configuration from. Default: ‘’

type –env:

str, optional

param –host:

The host IP address for the server. Default: ‘0.0.0.0’

type –host:

str, optional

param –port:

The port on which the server will listen. Default: ‘8080’

type –port:

str, optional

param –cml_dir:

CML folder to read the training scripts from, same as in Nova. Default: ‘cml’

type –cml_dir:

str, optional

param –data_dir:

Data folder to read the training scripts from, same as in Nova. Default: ‘data’

type –data_dir:

str, optional

param –cache_dir:

Cache folder where all large files (e.g., model weights) are cached. Default: ‘cache’

type –cache_dir:

str, optional

param –tmp_dir:

Folder for temporary data storage. Default: ‘tmp’

type –tmp_dir:

str, optional

param –log_dir:

Folder for temporary data storage. Default: ‘log’

type –log_dir:

str, optional

returns:

None

Example

>>> app.py --host 0.0.0.0 --port 53771 --cml_dir "/path/to/my/cml" --data_dir "/path/to/my/data" --cache_dir "/path/to/my/cache" --tmp_dir "/path/to/my/tmp"
